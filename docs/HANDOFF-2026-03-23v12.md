## What was done this session (2026-03-23, v12 session)

### Campaign v1.24 — 3-second pause before transition screen
When a level is won, the game now waits 3 seconds before firing `handleLevelEnd`
so the player sees the final grid state before the TransitionScreen appears.
Loss path (gameover overlay in-game) is unchanged.

### Campaign v1.23 — Campaign complete screen: stars in rows
All 40 stars were rendered in a single flat line, overflowing the card border.
Fixed to display in up to 4 rows of 10 stars each, left-to-right.

### Tablet v11.6 — Bonus moves logic mirrors campaign
Full architectural update: collapsed the two-pool system (`bonusMovePool` +
`bankedMoves`) into one persistent `bankedMoves` pool. Changes:
- Earned bonus moves go directly into `bankedMoves` (persistent across games)
- Prompt fires on win AND fail (old `!hasReachedTarget` gate removed)
- Prompt fires after bonus round ends if `bankedMoves > 0`
- "Use bonus moves": each swap draws 1 from `bankedMoves` (moves stays at 0)
- "End and carry moves forward": game ends, pool persists
- Header: removed 🏦 icon and 🎯 pool; single "Bonus moves: N" label
- Burst animation: `+1 🎯` → `+1 bonus move`

### Phone 418 v12.2 — Bonus moves logic mirrors campaign
Same set of fixes applied to the standalone HTML file. Phone 418 had already
done the architecture (v12.0 unified pool), but still had the old gates and
terminology. Changes:
- Prompt gate: removed `!hasReachedTarget` (fires on win AND fail)
- Bonus round end: check `bankedMoves > 0` before resolving as won
- `endGameCarryBanked`: now resolves as won or gameover based on target
  (previously always set gameover regardless)
- Unified `moves <= 0` resolve path across all branches
- Header: 🏦N → `"Bonus moves: N"`; burst `"+1 🏦"` → `"+1 bonus move"`
- Header button: removed 🏦 emoji count from label
- Prompt: context-aware title (`"🎯 Target reached!"` vs `"⚠️ Out of moves!"`),
  `"banked"` → `"bonus"` throughout, `"Use banked moves"` → `"Use bonus moves"`

## Terminology

### Confirmed terms

| Use | Avoid |
|---|---|
| **Bonus moves** | "banked moves", "saved moves", "extra moves" |
| **Victory round** | "bonus round" — to be migrated (see below) |
| **"End and carry moves forward"** | "End and save", other variants |
| **"Use bonus moves"** | "Use banked moves" |

### Planned rename: "bonus round" → "victory round"

The phase where a player has reached the target score with regular moves still
remaining is called the **bonus round** in code and UI. Rename to **victory round**
in a dedicated session — it touches many places and must be done in one pass.

**Scope:** UI strings, variable names (`bonusRoundActive`, `setBonusRoundActive`,
`startBonusRound`, `showBonusPrompt`, `bonusRoundScore`, `preBonusScore`),
comments throughout campaign file, and `docs/DESIGN.md`.
