## What was done this session (2026-03-22, v8 session)

### Campaign v1.11 ‚Äî time attack levels now use pause + prompt

Campaign time attack levels (3 and 6) previously auto-converted bonus moves one-at-a-time
on timer expiry (v1.10). v1.11 matches the arcade time attack v12 "Option C" pattern:
- When timer hits 0, score < target, and `campaignBonusMoves > 0`: timer pauses, modal appears
- "Use bonus move (+5s)" ‚Üí consume 1 move, set time to 5s, timer resumes
- "End level ‚Äî save moves" ‚Üí end level now, pool persists
- Prompt re-fires at each subsequent expiry until pool empty or target reached

Also fixed in v1.11: bonus moves earned during time attack levels (removed `!IS_TIME_ATTACK`
guard from bonus move award effect). Previously, 10k-point thresholds during time attack
were silently discarded. Now all level types award bonus moves correctly.

### Bug fix ‚Äî special+special swap now creates additional specials (all platforms)

When two specials are swapped, the code previously took an early return before calling
`findMatches`, so any 4+ tile match formed by the swap (adjacent regular tiles) was silently
lost and no additional special was placed.

Fix applied to all six platforms:
- Campaign v1.12
- Tablet v11.5
- Desktop v12.1
- Time Attack v12.1
- Phone 341 v12.1
- Phone 418 v12.1

**How the fix works:**
1. In `attemptSwap`, before scheduling `activateSpecialCombination`, run `findMatches` on
   the swapped grid and extract `connectedGroups` with ‚â•4 tiles (excluding pure-special groups).
2. Pass those groups as `additionalGroups` into `activateSpecialCombination`.
3. Inside `activateSpecialCombination`, after clearing `uniqueTiles`, iterate `additionalGroups`
   and place a special at the best cleared position ‚Äî same centroid + swap-position-preference
   logic as `removeMatches`.

Note: time attack v12.1 also calls `addTimeExtension` for each new special placed, matching
how it handles specials in `removeMatches`.
