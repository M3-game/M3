## What was done this session (2026-03-22)

### New docs: CLAUDE.md (root) and docs/DESIGN.md

`CLAUDE.md` now lives in the project root so Claude Code auto-loads it at every session
start. A copy is also at `docs/CLAUDE.md` for browsing.

`docs/DESIGN.md` is a new authoritative game design reference covering: bonus moves
mechanic (correct behavior, prompt logic), bonus round, time attack bonus conversion,
scoring constants, star thresholds, level unlock gates, known bugs, future work, and
terminology. **Read it before implementing any gameplay changes.**

Key decisions recorded in DESIGN.md:
- The term **"banked moves" is wrong** — use **"bonus moves"** throughout.
- The 🏦 icon is slated for removal in a future pass.
- The bonus moves prompt must fire at `moves = 0` **regardless of win or fail**.

### Arcade platforms — bonus moves pool (v12.0)

Phone 341, Phone 418 (standalone HTML), and Desktop all received the unified bonus
moves system:
- Earn 1 bonus move per 10k pts → persistent pool (per-platform localStorage key)
- When regular moves hit 0 and pool > 0: prompt fires whether target was reached or not
- "Use bonus moves" / "End and carry moves forward"
- localStorage keys: `match3_phone341_bankedMoves`, `match3_phone418_bankedMoves`,
  `match3_desktop_bankedMoves`

Note: Phone 418 is a standalone CDN-React HTML file — all UI is `React.createElement`
syntax, not JSX. Take care when editing it.

### Campaign v1.9 — bonus moves prompt fires on win AND fail

Previous logic gated the prompt on `!hasReachedTarget`, so it only appeared when the
player failed to reach the target score. Most players win before running out of regular
moves, meaning the prompt never fired in normal play.

Fix: removed the gate. Whenever `moves = 0` and `campaignBonusMoves > 0`, the prompt
fires. Prompt heading is context-aware: `"🎯 Target reached!"` vs `"⚠️ Out of moves!"`.
`endLevelCarryBanked` now adds the unused-specials bonus to the score before resolving.

### Campaign v1.10 — time attack bonus conversion: one at a time, stop on target

Previous code consumed the entire `campaignBonusMoves` pool at once on timer expiry.
Fix: convert one move per timer tick (+5s each), stop as soon as score ≥ target.
Remaining moves carry forward.

### Time Attack arcade v12 — bonus moves pool with prompted expiry (Option C)

The bonus move threshold tracking that was stubbed in v10.4 is now fully wired up:
- Earn 1 bonus move per 10k pts → persistent pool (`match3_timeattack_bankedMoves`)
- When timer hits 0 and score < target: timer **pauses**, modal prompt appears
- "Use bonus move (+5s)" → consumes one move, adds 5s, timer resumes
- "End game — save bonus moves" → game ends, pool persists
- Prompt re-appears at each expiry until pool is empty or target reached
- Pool shown in header when > 0 (🎯 counter)

This is **different from campaign time attack** (which is automatic). Time attack arcade
is player-prompted, one explicit choice per move.

## Known bugs / todos (carry from v6 + new)

### HIGH PRIORITY

1. **Time attack: star rating recorded too early** (campaign, bug)
   Stars are snapshotted the moment the timer hits 0, but tiles may still be falling
   and their matches haven't registered. Score is undercounted at the snapshot moment.
   Fix: wait for the board to fully settle (`isAnimating === false && combo === 0`)
   before evaluating the final score and awarding stars.
   File: `match3-v1.10-campaign-tablet.jsx`, timer expiry block.

### LOWER PRIORITY / FUTURE SESSIONS

2. **Transition screen missing fields** — spec calls for: moves used efficiency, best
   combo achieved, specials created, total bonus moves now banked. Star fanfare
   animation also absent.

3. **Game over screen missing fields** — best score this level, stars vs personal best,
   "Start Over" button (reset to L1, keep records).

4. **Campaign complete screen** — currently redirects to map; needs dedicated screen
   (total score, stars/40, replay prompt, achievements).

5. **Header** — "Level X of 8" indicator and running campaign total score not shown.

6. **Target scores** — all 8 levels still using placeholder targets; needs gameplay
   testing to calibrate.

7. **Time attack star thresholds** — L3 and L6 TBD pending testing.

8. **Supernova/Hypernova placement thresholds** — not calibrated for campaign board
   sizes / target scores.

9. **Time attack arcade timer dep array includes `score`** — the timer `useEffect` has
   `[gameState, score, highScore]` as deps, causing the interval to restart on every
   score change. This makes the 1-second tick inconsistent during active play. Low
   severity, pre-existing. File: `match3-v12-desktop-timeattack.jsx`.

10. **`bonusMoveFlashPendingRef` in time attack is vestigial** — declared and reset in
    `restartGame` but never used. Left in place to avoid unintended side effects.

11. **Level 7 gate: raise to 4★** — future change, raise from 3★ to 4★ on L5 or L6.
    Star total thresholds unchanged. Transition screens before L5, L6, L7 should
    explain unlock requirements. See `docs/DESIGN.md`.

12. **Remove "banked" terminology** — rename `campaignBonusMoves` and related variables
    to drop "banked"; remove 🏦 icon. See `docs/DESIGN.md`.
