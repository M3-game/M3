## What was done in this session (2026-03-21)

### Campaign v1.5 — cascade popups in activateSpecialCombination
When two specials are swapped together, any specials swept up by the resulting blast now each get their own score popup with a 1.5x cascade multiplier, staggered 300ms apart. Previously all cascade points were lumped into the main combo popup silently.

### Campaign v1.6 — banked moves UX
When regular moves hit 0 and bonus moves remain, a prompt appears:
- **"Use banked moves"** — player keeps playing; 🏦 counter decrements with each move (regular moves counter stays at 0); "End and carry moves forward" button persists in the header.
- **"End and carry moves forward"** / **"End"** (level 8) — level ends, all remaining 🏦 carries to next level.

### Campaign v1.7 — cascade scoring bug fix
Identified and fixed a bug introduced in v1.5: the while-loop in `activateSpecialCombination` pre-added each cascade special's key to `processedInCombination` before passing that set to `activateSpecialTile`. Since `activateSpecialTile` checks `alreadyCleared.has(posKey)` on entry, it returned empty results immediately — zero cascade score, no popup, and no further tile clearing. Fix: removed the pre-add; `activateSpecialTile` adds the key internally (mutates the shared set by reference), so deduplication still works.

### Campaign v1.8 — unified bonus/banked moves pool
Removed the distinction between in-level bonus moves (`bonusMovePool` / 🎯) and carried banked moves (`campaignBonusMoves` / 🏦). From the player's perspective these are the same thing. Now:
- Bonus moves earned during play (every 10k pts) go directly into 🏦.
- One counter, one number — grows during play, carries across levels.
- The separate 🎯 display and old "Use extra moves / Save moves" prompt are removed.
- "Use banked moves" prompt and "End and carry moves forward" header button use the unified 🏦 pool.

## Known bugs / todos (carry to next session)

### HIGH PRIORITY

1. **Time attack: star rating recorded too early** (bug)
   Stars are snapshotted the moment the timer hits 0, but tile-match animations and combos may still be running. Score can increase after the snapshot, bumping the player into a higher star tier that never gets saved or shown on the map.
   Fix: defer the star/score snapshot until `isAnimating === false && combo === 0`, or route time attack wins through the existing `handleLevelEnd` / `setGameState('won')` path which already runs post-animation.
   File: `match3-v1.8-campaign-tablet.jsx`, timer expiry block inside `setTimeLeft(prev => ...)`.

2. **Time attack: banked-move expiry extension spends all moves at once** (design improvement)
   Current: on timer expiry, if target not reached and 🏦 > 0, all banked moves are consumed immediately (+5s each).
   Better: add one move's worth of time at a time (+5s), check score after each addition, stop as soon as target is reached. Avoids over-spending moves when only 1–2 extra seconds were needed.
   File: `match3-v1.8-campaign-tablet.jsx`, timer expiry block inside `setTimeLeft(prev => ...)`.

## Future feature ideas

### Banked moves for arcade platforms
The campaign's unified bonus/banked moves system (v1.8) could be ported to all arcade platforms (tablet, desktop, time attack, phone 341, phone 418). Arcade games are currently self-contained single sessions, but with banked moves a player could carry bonus moves forward from one game to the next — adding a meta-progression layer to the arcade modes. Implementation would follow the same pattern as campaign v1.8: bonus moves earned per 10k pts go into a persistent 🏦 counter (localStorage), and when regular moves hit 0 the player is offered "Use banked moves" or "End and carry moves forward."

### Dev / Debug slow-motion mode
A toggleable Dev mode that slows down all gameplay — moves, combos, and cascades — so each step can be observed. Features:
- Adjustable speed (slower, paused mid-combo or mid-cascade)
- "Stop at any moment" — pause button that freezes the game mid-animation
- Score popup for every individual score event, showing exactly how it was generated (e.g. "Line clear · 12 tiles · 30pts each = 360", "CASCADE x1.5 · Bomb · 750 = 1125")
- Useful for debugging scoring bugs, cascade chains, and multiplier stacking

## Versioning convention (mandatory)

Before every change — no exceptions, even one-liners:
1. Copy current file to `platforms/<platform>/archive/`
2. Create new versioned file (e.g. v1.8 → v1.9)
3. Update `src/entry-campaign.jsx` (or `src/main.jsx`) to point to new file
4. Update `index.html` version label
5. Update version comment block at top of new file

See `docs/versioning-convention.md` for full checklist.
