## What was done in this session (2026-03-20 v3)

### Bug fixes
- **Phone-418 blank screen**: unclosed header div in React.createElement chain — fixed by adding missing `)` before canvas wrapper.
- **Arcade tablet v11.3 → v11.4**: game-end effect / bonus-move effect order was wrong; game-end now runs first so the bail-out check sees the stale `bonusMoveThresholdRef` and doesn't fire on the same turn as a bonus-move award.
- **v11.3 archive recovery**: v11.3 had been overwritten in place; recovered from `git show 09c4f95:...` and properly archived.

### Campaign UX — v1.2
- Sequential flow: win → TransitionScreen → next level (map is optional, accessed from transition or on loss)
- Transition screen: level complete, 1–5 stars, score, bonus moves earned, timed-mode callout for next level, Next Level / View Map buttons
- Loss → retry overlay (Retry / View Map); no forced map on loss
- Tap-to-start overlay for time attack levels
- LevelSelectScreen: future/locked levels shown as faint dashed cards

### Campaign cascade fix — v1.3
- When two specials are combined (e.g. line + bomb), any specials swept up by the resulting clear were silently deleted instead of activating. Fixed with a while-loop cascade in `activateSpecialCombination`.
- Arcade platforms (tablet v11.4, desktop v11, time attack v11, phone-341 v11, phone-418 HTML) confirmed already had full cascade logic — no changes needed.

### Campaign time attack extensions — v1.4
Two mechanics implemented:

**Arcade mechanic (during play)** — mirrors arcade time attack:
- +5s per combo x5+
- +5s per special created (line, bomb, cross, supernova, hypernova)
- +5s per 5,000 points scored
- Capped at 15s per player action (`TIME_EXTENSION_CAP_PER_TURN`)
- Green floating popups appear at top of screen

**Spec mechanic (timer expiry safety net)**:
- If timer hits 0 before target is reached AND player has banked campaign bonus moves, all banked moves are consumed at once, adding +5s each
- One-time per level; popup shows how many seconds were added

## Known bugs / todos (carry to next session)

### Campaign — high priority

1. **Time attack: improve banked-move extension mechanic** (design improvement)
   Current: on timer expiry, all banked moves are consumed at once (+5s × N moves).
   Better: add one move at a time (+5s), check score after each, stop as soon as
   target is reached. This avoids over-spending moves when only 1–2 more seconds
   were needed.
   File: `match3-v1.4-campaign-tablet.jsx`, timer expiry block in `setTimeLeft(prev => ...)`.

2. **Time attack: star rating recorded too early** (bug)
   Stars are calculated and saved at the moment the timer expires, but tile-matching
   animations and combos may still be running. Score can increase after the star
   snapshot, bumping the player into a higher star tier that never gets recorded or
   shown on the map.
   Fix: delay the star/score snapshot until all animations settle (i.e. after
   `isAnimating` is false and `combo === 0`), or record the final score in
   the existing `handleLevelEnd` / `setGameState('won')` path which already
   runs post-animation.

### Campaign — lower priority / future sessions

3. **Transition screen missing fields** — spec calls for: moves used efficiency,

   Star fanfare animation also missing.

5. **Campaign complete screen** — currently redirects to map; needs dedicated screen

7. **Target scores** — all 8 levels still using placeholder targets; needs gameplay

9. **Supernova/Hypernova placement thresholds** — retained from arcade, not calibrated for campaign board sizes / target scores.

## Versioning convention (mandatory)

Before every change — no exceptions, even one-liners:
1. Copy current file to `platforms/<platform>/archive/`
2. Create new versioned file (e.g. v1.4 → v1.5)
3. Update `src/entry-campaign.jsx` (or `src/main.jsx`) to point to new file
4. Update `index.html` version label
5. Update version comment block at top of new file

See `docs/versioning-convention.md` for full checklist.
